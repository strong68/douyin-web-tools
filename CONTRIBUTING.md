# Contributing

欢迎继续改进这个仓库。

这个项目当前更看重两件事：

- 代码保持清晰、稳定、容易检查
- 改动在真实网页环境里更可靠

如果你愿意参与，我们非常欢迎 issue、建议、文档修正和代码改进。

## Before You Start

提交改动前，建议先确认：

- 这个问题是否已经在 README 中说明过
- 这个问题是否和抖音页面结构变化有关
- 这个改动是否会影响现有 4 个子项目中的参数风格或使用方式

如果你的改动会改变命令行参数、输出格式或登录流程，请在说明里明确写出来。

## Good Contribution Areas

比较适合继续改进的方向：

- 提升页面改版后的兼容性
- 提升异常恢复能力
- 优化元素定位策略
- 补充自动化测试和回归测试
- 改进日志、错误提示和输出格式

## Coding Guidelines

请尽量遵守当前仓库的风格：

- 保持结构简单，不要为了抽象而抽象
- 优先保证可读性，其次才是“写得很巧”
- 尽量保持每个子项目职责单一
- 尽量复用现有参数命名风格

当前仓库已经统一使用的常见参数包括：

- `--wait-for-login`
- `--max-count`
- `--output-dir`
- `--json`

## Pull Request Tips

提交 PR 时，建议说明：

- 你修了什么问题
- 影响了哪个子项目
- 是否修改了命令行参数或输出格式
- 你做了哪些验证

如果可以，尽量附上：

- 运行命令
- 关键输出
- 出现问题时的报错信息

## Issue Tips

提 issue 时，建议包含：

- 使用的是哪个子项目
- 使用的命令
- Python 版本
- 浏览器环境
- 是否已登录抖音
- 预期行为
- 实际行为

## Notes

- 本项目依赖真实网页结构，页面改版后可能需要同步调整
- 某些问题可能不是代码 bug，而是登录状态、权限、地区或页面变化导致

感谢你的改进和交流。
