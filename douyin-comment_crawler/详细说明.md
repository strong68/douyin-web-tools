# douyin-comment-crawler 详细说明

> 文件位置：`douyin_comment_crawler/douyin_comment_crawler.py`

---

## 一句话概述

根据视频 id 爬取该视频下的全部评论和回复，评论经过去除 emoji、表情包符号等噪音处理，最终以结构化 JSON 格式保存到本地文件，同时附带视频信息和作者信息。

---

## 核心流程

```
输入 video_id
    ↓
打开视频页，监听详情接口 → 获取视频基本信息（标题、发布时间、标签、作者信息）
    ↓
进入评论列表页，监听评论接口
    ↓
滚动评论区，每次滚动后点击"展开回复"按钮，监听并收集评论和回复
    ↓
评论去重、文本清洗（去除 emoji、表情符号）
    ↓
达到 max_comments 数量或翻到底部时停止
    ↓
结果以 JSON 格式保存到 {video_id}_comments.json
```

---

## 类常量

| 常量 | 值 | 说明 |
|---|---|---|
| `COMMENT_COUNT_THRESHOLD` | `1000` | 评论总数超过此值时将被跳过（可自定义阈值） |
| `COMMENT_LIST_XPATH` | 很长一串 XPath | 评论区列表容器的定位符 |
| `COMMENT_SCROLL_XPATH` | 含 `parent-route-container` 的 XPath | 评论区的滚动区域定位符 |
| `COMMENT_END_XPATH` | 同上，含 `{index}` 占位符 | 评论区"暂无更多评论"提示元素的定位符 |
| `COMMENT_EXPAND_CLASSES` | `{'FgYRerj2', 'J_mtqkIZ comment-reply-expand-btn'}` | "展开回复"按钮的 class 集合 |
| `COMMENT_REPLY_PANEL_XPATH` | 含 `eBcBNgnP` 的 XPath | 展开回复后面板容器的 XPath |
| `NO_MORE_COMMENT_TEXT` | `'暂时没有更多评论'` | 翻到底部的提示文字 |
| `DEFAULT_OUTPUT_DIR` | 脚本所在目录 | 默认输出目录 |

## 方法详解

### `_parser() -> argparse.ArgumentParser`

```
--video-id       视频 id（必填）
--max-comments   最多抓取评论数，默认 100
--output-dir     保存目录，默认脚本所在目录
```

### `_safe_close_tab(tab) -> None`

安全关闭标签页，捕获异常防止崩溃。

### `_should_skip_comment_crawl(total_count, threshold=1000) -> bool`

当评论总数超过阈值时返回 `True`，用于判断是否跳过爬取（评论过多时爬取成本高）。

### `_response_body_to_dict(response) -> Dict`

将监听接口返回的 response 对象标准化为 dict：
- 已有 dict 直接返回
- 字符串则 `json.loads`
- 其他类型返回空 dict

### `_is_expand_button(button) -> bool`

判断一个按钮元素是否是"展开回复"按钮（class 命中 `COMMENT_EXPAND_CLASSES`）。

### `_clean_comment_text(text) -> str`

评论文本清洗步骤：
1. 用正则删除所有 emoji（unicode 表情范围）
2. 删除 `[xxx]` 格式的表情包符号
3. 合并多个空白字符为空格
4. 若剩余文本不含任何中/英文/数字，返回空字符串（无效评论丢弃）

### `_save_json(path, payload) -> None`

将 payload 以 UTF-8 编码的格式化 JSON 写入文件。

### `_count_comments(comments_dict) -> int`

遍历 `comments_dict['comments']` 字典，按 comment_id 去重计数，包含主评论和所有回复。

### `_organize_comments(comments_dict, new_comments_list, max_comments) -> Dict`

将新获取的评论列表组织到 `comments_dict` 中：
- `reply_id == '0'` → 写入 `comments` 主字典
- 否则 → 追加到对应父评论的 `reply_comments` 列表（去重）
- 支持嵌套：回复的回复同样递归处理
- 达到 `max_comments` 立即停止

### `_persist_result(video_id, comments_data) -> str`

将 `comments_data` 保存为 `{video_id}_comments.json`，返回文件路径字符串。

### `crawl_video_comments(video_id, max_comments=100, comment_threshold=None) -> Dict`

完整爬取流程：

**第一步：获取视频信息**（新建标签页）
- 打开 `https://www.douyin.com/video/{video_id}`
- 监听 `aweme/detail/` 接口
- 从 `aweme_detail` 提取：标题、发布时间、作者 uid、sec_uid、标签、评论总数

**第二步：获取作者信息**（若有 sec_uid，新建标签页）
- 打开作者主页
- 监听 `web/user/profile/other/` 接口
- 提取作者昵称、IP 属地、城市、区县

**第三步：爬取评论**
- 新建标签页进入视频页
- 监听 `comment/list/` 和 `comment/list/reply/` 接口
- 初始化 `comment_summary`
- 进入翻页循环：
  - 等待评论接口响应，收集评论
  - 对每个评论，点击"展开回复"按钮，监听回复接口
  - 滚动评论区
  - 检测"暂无更多评论"提示，到达 `max_comments` 或翻到底则退出
- 更新 `collected_count`

**第四步：保存**
- 调用 `_persist_result` 写入 JSON
- 返回完整数据结构

### `close() -> None`

安全关闭浏览器。优先尝试 `quit()`，失败则尝试 `close()`。

## 使用示例

### 命令行

```bash
# 抓取默认 100 条评论
python douyin_comment_crawler.py --video-id 7431772994717879567

# 抓取 200 条，保存到指定目录
python douyin_comment_crawler.py --video-id 7431772994717879567 --max-comments 200 --output-dir D:\comments
```

### 代码调用

```python
from douyin_comment_crawler import DouyinCommentCrawler

crawler = DouyinCommentCrawler()
result = crawler.crawl_video_comments(
    video_id='7321234567890123456',
    max_comments=200,
)
print(f"共抓取 {result['comment_summary']['collected_count']} 条评论")
crawler.close()
```

### 与搜索组合：搜索 → 抓评论

```python
from douyin_searcher import DouyinSearcher
from douyin_comment_crawler import DouyinCommentCrawler

searcher = DouyinSearcher(max_count=5)
crawler = DouyinCommentCrawler()

result = searcher.search('搞笑视频')
for video in result['videos']:
    crawler.crawl_video_comments(video_id=video['aweme_id'], max_comments=50)

searcher.close()
crawler.close()
```
